# SBIM-TC-WEB 빌드 & 배포 가이드

다른 PC(Cursor 동일 환경)에서 이 문서만 보고 복사·붙여넣기로 빌드·배포할 수 있도록 정리했습니다.

---

## 한 번에 복붙 (처음 셋팅)

Cursor에서 **터미널 열기**(Ctrl+`) 후, 프로젝트 루트에서 아래 **한 블록 통째로** 복사해 붙여넣고 Enter.

```powershell
node scripts/first-time-setup.cjs
```

끝나면 `npm run dev:all` 로 로컬 실행, `npm run deploy:iis` 로 배포용 빌드 가능.

---

## 0. 필요한 것 (한 번만)

- **Node.js 20** (LTS 권장). 프로젝트에 `.nvmrc` 있으면 `nvm use` 또는 Node 20 설치.
- **npm** (Node 설치 시 포함)
- **Cursor** (또는 VSCode)에서 이 프로젝트 열기

---

## 1. 처음 프로젝트 받았을 때 (한 번만 실행)

프로젝트를 클론하거나 압축 풀었다면, **프로젝트 루트 폴더**에서 아래를 순서대로 실행하세요.

### 1-1. Node 버전 확인 (20 권장)

```powershell
node -v
```
`v20.x.x` 나오면 OK. 아니면 [Node 20 LTS](https://nodejs.org/) 설치 후 다시 확인.

### 1-2. 의존성 설치

```powershell
npm install
```

### 1-3. (선택) 한 번에 설정까지 끝내기

아래 스크립트로 의존성 설치 + 설정 확인까지 한 번에 할 수 있습니다.

```powershell
node scripts/first-time-setup.cjs
```

이후 **로컬 실행**은:

```powershell
npm run dev:all
```

브라우저에서 `http://localhost:5173` 접속.

---

## 2. 배포용 빌드 (배포할 때마다)

서버(IIS 등)에 올리기 전에 **반드시** 프로젝트 루트에서 아래 중 하나를 실행하세요.

### 2-1. 빌드 + 배포 패키지 준비 (권장, 한 줄)

```powershell
npm run deploy:iis
```

- 프론트: `VITE_BASE_PATH=SBIM-TC-WEB` 로 빌드 → `publish-iis/dist`
- 서버: `server/` → `publish-iis/server/` 복사 후 `npm install --omit=dev`
- **결과**: `publish-iis` 폴더 전체가 서버에 복사할 최종 패키지입니다.

### 2-2. 단계 나눠서 하고 싶을 때

```powershell
npm run build:iis
```
→ 프론트만 서브경로로 빌드하고 `dist` 를 `publish-iis/dist` 에 복사.

```powershell
npm run prepare-publish-iis
```
→ 서버 소스를 `publish-iis/server` 에 복사하고, 그 안에서 `npm install --omit=dev` 실행.

---

## 3. 서버에 복사하기

`publish-iis` 폴더 **전체**를 서버에서 앱이 돌아갈 위치에 그대로 복사합니다.

### 3-1. 수동 복사

- **대상 폴더 예**: `E:\WebApps\sbim-tc-web` (서버 PC 경로는 환경에 맞게 변경)
- **복사할 것**: `publish-iis` **안의 모든 것** (폴더 통째로 복사해도 됨)
  - `web.config`
  - `dist/` (프론트 UI)
  - `server/` (Node API + node_modules 포함)
- **주의**: 기존 서버 폴더를 **지우지 말고** 덮어쓰기.  
  PostgreSQL은 서버에 별도로 두며, `server/uploads/` 는 배포 패키지에 없으므로 서버에 있으면 그대로 유지됩니다.

### 3-2. Cursor/터미널에서 서버 경로로 한 번에 복사 (Windows)

서버 경로를 알고 있으면 아래처럼 실행하면 **빌드 + publish-iis 준비 + 해당 경로로 복사**까지 한 번에 됩니다.

```powershell
node scripts/deploy-iis-to-server.cjs E:\WebApps\sbim-tc-web
```

경로는 실제 서버 배포 경로로 바꾸세요.  
같은 PC가 아니라면 `publish-iis` 폴더만 USB/공유폴더로 옮긴 뒤 서버 PC에서 같은 구조로 붙여넣으면 됩니다.

## 5. 서버(IIS) 쪽 설정

서버 PC에서 **최초 1회**만 하면 되는 것들입니다.

- **Node.js** 설치 (LTS, 기본 경로 권장)
- **iisnode** 설치: https://github.com/Azure/iisnode/releases
- **IIS URL Rewrite** 모듈 설치
- IIS에서 사이트/앱 풀 설정 후 `web.config` 가 있는 폴더를 해당 사이트 루트로 지정

자세한 단계는 **publish-iis** 안의 다음 파일을 참고하세요.

- `publish-iis/IIS-SETUP.txt`
- `publish-iis/서버에 복사하기 전에 꼭 읽기.txt`

---

## 6. 환경 변수 (선택)

- **프론트**: `.env` 에 `VITE_TRIMBLE_CLIENT_ID` 등 필요 시 설정 (없으면 코드 기본값 사용)
- **서버**: `TRIMBLE_CLIENT_ID`, `TRIMBLE_CLIENT_SECRET` 은 서버에서 토큰 교환 시 사용. 없으면 코드 기본값 사용.
- **배포 경로가 / 가 아닌 서브경로** (예: `/SBIM-TC-WEB`) 인 경우, 빌드 시 `VITE_BASE_PATH=SBIM-TC-WEB` 가 이미 `build:iis` / `deploy:iis` 에 들어가 있으므로 별도 설정 없이 됩니다.

## Nginx로 배포할 때

IIS 대신 **Nginx**로 배포하려면 아래처럼 하면 됩니다.

### 빌드 (한 줄)

```powershell
npm run deploy:nginx
```

→ **publish-nginx** 폴더에 `dist/`, `server/` 가 준비됩니다.

### 서버에서 할 일

1. **publish-nginx** 폴더 전체를 서버에 복사.
2. **Node 실행**: 서버에 PM2 설치 후, `publish-nginx` 폴더에서  
   `pm2 start ecosystem.config.cjs`  
   (설정: `publish-nginx/ecosystem.config.cjs` — PORT, BASE_PATH 수정 가능)
3. **Nginx 설정**: `publish-nginx/nginx.conf.example` 내용을 참고해  
   `/SBIM-TC-WEB/` 요청을 `http://127.0.0.1:5001` 로 프록시하도록 설정.
4. Nginx 리로드 후 `http://도메인/SBIM-TC-WEB/` 로 접속 확인.

자세한 단계는 **publish-nginx/README-Nginx배포.txt** 를 보세요.
