# 다른 사람이 접속할 수 있게 배포하기

로컬(localhost)이 아닌 **인터넷 주소**로 배포하면, 링크만 보내서 다른 사람도 접속할 수 있습니다.

---

## 방법 1: Render (무료, 추천)

1. **GitHub에 코드 올리기**  
   - 이미 했다면 생략. 없다면: `git push origin main` 으로 `alsgur0133/SBIM-TC-WEB` 등에 푸시.

2. **Render 가입 및 서비스 만들기**  
   - https://dashboard.render.com 가입 (GitHub 로그인 가능)  
   - **New +** → **Web Service**  
   - **Connect repository**에서 `SBIM-TC-WEB` 저장소 선택  
   - 아래처럼 설정:
     - **Build Command:** `npm install && npm run build && cd server && npm install --omit=dev && cd ..`
     - **Start Command:** `node server/index.js`
     - **Environment** (선택):
       - `NODE_ENV` = `production`
       - `VITE_API_URL` = 비워두기(빈 값). 그러면 배포된 같은 주소에서 `/api` 사용
   - **Create Web Service** 클릭

3. **배포 완료**  
   - 5~10분 정도 지나면 빌드가 끝나고, **자동으로 URL**이 생깁니다.  
   - 예: `https://sbim-tc-web-xxxx.onrender.com`  
   - 이 주소를 다른 사람에게 보내면, 같은 앱에 접속할 수 있습니다.

4. **무료 제한**  
   - 무료 플랜은 15분 동안 접속이 없으면 서버가 잠들었다가, 다음 접속 시 다시 깨웁니다(첫 로딩이 조금 느릴 수 있음).

---

## 방법 2: Railway

1. https://railway.app 가입 후 **New Project** → **Deploy from GitHub repo**  
2. `SBIM-TC-WEB` 저장소 선택  
3. **Settings**에서:
   - **Build Command:** `npm install && npm run build && cd server && npm install --omit=dev && cd ..`
   - **Start Command:** `node server/index.js`
4. **Deploy** 후 생성된 **공개 URL**을 다른 사람에게 공유하면 됩니다.

---

## 방법 3: 내 PC에서 항상 켜두기 (테스트용)

- PC에서 `npm run build` 후 `npm run start:prod` 로 서버를 켜 두고,
- **공유기/방화벽**에서 포트(5001) 포워딩 + **공인 IP** 또는 **ngrok** 같은 터널로 임시 주소를 만들 수 있습니다.  
- 보안·안정성 때문에 **실제 서비스는 Render/Railway 같은 호스팅 사용을 권장**합니다.

---

## 배포 후 확인

- 배포된 URL(예: `https://sbim-tc-web-xxxx.onrender.com`)로 들어가서:
  - 로그인(이메일: sa, 비밀번호: 1234)이 되는지,
  - 메뉴/기능이 정상인지 확인하면 됩니다.
- **같은 주소**를 다른 사람에게 보내면, 그들도 브라우저에서 접속할 수 있습니다.
