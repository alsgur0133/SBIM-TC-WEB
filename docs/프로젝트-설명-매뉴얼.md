# SBIM TC Web (BRACE) — 프로젝트 설명 매뉴얼

설계·물량·모델을 프로젝트 단위로 관리하는 **웹 애플리케이션**입니다. 상단 브랜드명은 **BRACE**로 표시됩니다.

---

## 1. 목적과 사용자

| 구분 | 설명 |
|------|------|
| **목적** | 프로젝트별 설계 일정·도서·검토·모델(IFC 등)·물량(엑셀)을 등록·조회·집계하고, Trimble Connect와 연동해 모델을 확인할 수 있게 함 |
| **주 사용자** | 프로젝트 관리자, 관리자, 일반 참여자(권한에 따라 메뉴 제한) |
| **형태** | 브라우저에서 동작하는 SPA(React) + REST API(Node) |

---

## 2. 기술 구성 요약

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React 18, TypeScript, Vite 5, React Router 6 |
| UI·차트 | 자체 CSS(`App.css`), Recharts |
| 3D/IFC | three.js, web-ifc, web-ifc-three |
| 백엔드 | Node.js, 단일 진입 `server/index.js`(Express 스타일 라우팅) |
| DB | PostgreSQL (`DATABASE_URL`) |
| 파일 | 업로드(도서, 모델, 물량 엑셀 등), Multer 등 |
| 외부 연동 | Trimble Connect(OAuth/PKCE, Workspace API 패키지) |

개발 시 일반적으로 **프론트 5173**, **API 5001**을 사용하며, Vite가 `/api`를 API 서버로 프록시합니다.

---

## 3. 화면(메뉴) 구조

로그인 후 사이드바 기준입니다. **프로젝트 선택**과 **설계 차수·리비전**은 헤더에서 선택하며, 설계/물량 관련 화면에서 공통으로 사용됩니다.

| 메뉴 | 경로 | 설명 |

| 프로젝트 목록 | `/projects` | 프로젝트 CRUD(권한 필요) |

| 설계도서 관리 | `/design-doc` | 도서 파일 업로드·목록·다운로드 등 |

| 모델 뷰어 | `/trimble-viewer` | Trimble Connect 연동 안내 및 별도 창에서 보기 |

| 메뉴 | 경로 | 설명 |

## 4. 권한(역할) 개요

- **관리자**, **프로젝트 관리자**: 프로젝트·사용자·물량파일 등록/삭제 등 관리 기능에 접근 가능한 항목이 많습니다.
- **일반 사용자**: 조회 위주이며, 화면에 따라 버튼이 비활성화되거나 메뉴가 제한될 수 있습니다.

실제 제한은 `server/index.js`의 API 권한 체크와 각 페이지의 `canManage` 등과 연동됩니다.

---

## 5. 설계 차수·리비전과 데이터 연동

1. **설계일정 관리**에서 차수(예: 1차 설계)와 리비전(예: Rev 0)을 만듭니다.
2. 헤더 드롭다운에서 **같은 프로젝트**의 차수·리비전을 선택합니다.
3. 설계도서, 모델, 물량 등은 선택된 **리비전(`design_revision_id`)**에 묶여 저장·조회됩니다.

차수/리비전을 추가·삭제한 뒤 헤더가 바로 안 바뀌면 브라우저 새로고침 전에, 최신 코드에서는 설계일정 화면에서 저장 시 **전역 컨텍스트가 갱신**되도록 연동되어 있습니다.

## 6. 물량 엑셀(참고)

물량파일 등록 시 서버가 시트 형식에 따라 파싱합니다. 대표적으로:

- **부재별산출서** 형(층·부호·명칭·규격 등 고정 열)
- **반입 일정** 형(부재번호·총물량 등)
- **부재별 집계표**(ID·도면번호·총물량·SIZE·이형철근/부속 열 등)

집계표 화면에서는 **명칭 매핑·규격 매핑**으로 콘크리트/거푸집/철근 분류와 맞춥니다. 자세한 규칙은 물량파일 등록 화면 안내와 서버 `parseQuantityExcelSheet` 주석을 참고하세요.

---

## 7. Trimble Connect 연동(요약)

- Trimble 개발자 포털에서 앱 등록 후 **Client ID**, **Redirect URI**, **앱 이름(scope)** 등을 발급받습니다.
- 루트 `.env`(및 Vite용 `VITE_*`)에 설정합니다. **예시 키 이름**은 저장소의 `.env.example`을 보고, **실제 비밀 값은 저장소에 커밋하지 마세요.**
- 상세 흐름은 `docs/TRIMBLE-CONNECT-PROJECT-SYNC.md` 등을 참고합니다.

---

## 8. 로컬에서 실행하기

1. Node.js 설치(프로젝트에 `.nvmrc`가 있으면 해당 버전 권장).
2. 루트에서 `npm install`.
3. `server` 폴더에서도 `npm install`(및 Windows에서 네이티브 모듈이 필요하면 `npm run rebuild:server` 등).
4. `.env.example`을 참고해 `.env` 작성(필요한 항목만).
5. 터미널에서:
   - `npm run dev:all` — API(5001) + Vite(5173) 동시 실행
   - 또는 `npm run server`와 `npm run dev`를 각각 실행

브라우저에서 개발 서버 주소(기본 `http://localhost:5173`)로 접속합니다.

## 9. 빌드·배포

| 스크립트(예) | 용도 |
|--------------|------|
| `npm run build` | 프론트 `dist` 생성 |
| `npm run deploy:iis` | IIS 배포용 산출물 준비(서브경로는 환경 변수로 지정) |
| `npm run deploy:nginx` | Nginx 배포용 산출물 준비 |

서브경로 배포 시 `VITE_BASE_PATH` 등은 `vite.config.ts` 및 `docs/빌드배포-다른프로젝트에-적용하기.md`를 따릅니다.  
IIS 설치·iisnode·HTTPS 등은 저장소 루트의 `IIS서버-설치방법.txt`, `publish-iis` 안내 파일을 참고합니다.
