# 같은 빌드·배포 방식 적용 가이드

Cursor에서 Vite(React) + Node 서버로 개발한 웹 프로젝트를, **IIS + 서브경로**로 빌드·배포하는 방법을 **다른 사람이 자기 프로젝트에 그대로 적용**할 수 있도록 정리한 문서입니다.  
이 문서와 아래 스크립트만 전달하면, 받는 사람이 자기 프로젝트에 복사·수정해서 쓸 수 있습니다.

> **비전공자·초기 설정이 전혀 없는 프로젝트**는 먼저 **[IIS배포-처음부터-초보자가이드.md](./IIS배포-처음부터-초보자가이드.md)** 를 순서대로 따라 하면 됩니다. (Node 설치, 폴더 확인, `deploy:iis`까지)  
> **복사만 하면 되는 최소 묶음**은 저장소 루트의 **`iis-deploy-kit/`** 폴더(`README-복사방법.txt` 참고)입니다.

---

## 0. `npm run build`만 했는데 `publish` 폴더가 안 생길 때

**정상입니다.** Vite 기본 빌드는 **`dist` 폴더만** 만듭니다. **`publish-iis`(또는 publish)는 자동 생성되지 않습니다.**

다른 프로젝트에 이 문서의 **`scripts/build-for-iis.cjs`**, **`scripts/prepare-publish-iis.cjs`** 를 복사하고 `package.json`에 `build:iis` / `prepare-publish-iis` / `deploy:iis` 를 추가한 뒤 **`npm run deploy:iis`** 를 실행해야 `publish-iis` 가 채워집니다.

**스크립트 없이 수동으로 만들려면** (구조가 같을 때):

1. `vite build` (서브경로면 `VITE_BASE_PATH=앱이름` 넣고 빌드) → `dist` 생성
2. 빈 폴더 `publish-iis` 만들기
3. `dist` 전체 → `publish-iis/dist` 로 복사
4. `server` 폴더(소스) → `publish-iis/server` 로 복사 (`node_modules`는 제외하거나 복사 후 삭제)
5. `publish-iis/server` 에서 `npm install --omit=dev`
6. 이 문서 **5절**의 `web.config` 를 `publish-iis/web.config` 로 저장

이후 `publish-iis` 전체를 IIS 물리 경로에 넣으면 됩니다.

---

## 1. 이 방식 요약

| 항목 | 내용 |
|------|------|
| 프론트 | Vite 빌드. 서브경로(예: `/MY-APP/`)로 배포 시 `base` 설정 후 빌드 |
| 서버 | Node(Express 등) `server/` 폴더. API + 정적 파일(dist) 서빙 |
| 배포 | `publish-iis` 폴더에 `dist` + `server`(node_modules 포함) + `web.config` 넣어서 IIS에 복사 |
| IIS | iisnode로 `server/index.js` 실행, 모든 요청을 Node가 처리 |

---

## 2. 전제 조건 (자기 프로젝트 구조)

- **프론트**: Vite 사용 (또는 `npm run build` 로 `dist` 생성)
- **서버**: 프로젝트 루트 아래 `server/` 폴더, 진입 파일 예: `server/index.js`
- **배포 경로**: 서브경로(예: `http://도메인/MY-APP`) 또는 루트(`/`)

---

## 3. 자기 프로젝트에 적용하는 단계

### 3-1. 서브경로 이름 정하기

예: `MY-APP` (IIS에서 쓰는 가상 디렉터리/앱 이름과 맞추면 됨)

### 3-2. Vite에 base 설정

`vite.config.ts` (또는 `vite.config.js`)에서:

```ts
// 서브경로 배포 시
const basePath = process.env.VITE_BASE_PATH  // 예: 'MY-APP'
const base = basePath ? `/${basePath.replace(/^\/|\/$/g, '')}/` : '/'
export default defineConfig({
  base,
  // ... 나머지 설정
})
```

### 3-3. 스크립트 폴더와 파일 만들기

프로젝트 루트에 `scripts` 폴더를 만들고, 아래 **4. 복붙용 스크립트**를 그대로 붙여넣어 저장합니다.  
파일명:

- `scripts/build-for-iis.cjs`
- `scripts/prepare-publish-iis.cjs`

각 파일 **맨 위**에 있는 `BASE_PATH_NAME`, `SERVER_DIR_NAME` 만 자기 프로젝트에 맞게 수정합니다.

### 3-4. package.json에 스크립트 추가

```json
"scripts": {
  "build:iis": "node scripts/build-for-iis.cjs",
  "prepare-publish-iis": "node scripts/prepare-publish-iis.cjs",
  "deploy:iis": "npm run build:iis && npm run prepare-publish-iis"
}
```

### 3-5. publish-iis 폴더와 web.config

- 프로젝트 루트에 `publish-iis` 폴더 생성.
- **5. 복붙용 web.config** 내용을 `publish-iis/web.config` 로 저장.
- 서버 진입 파일이 `server/index.js` 가 아니면 `web.config` 안의 `server/index.js` 를 해당 경로로 수정.

### 3-6. 배포 실행

```powershell
npm run deploy:iis
```

실행 후 `publish-iis` 안에 `dist/`, `server/`(node_modules 포함), `web.config` 가 있으면, 이 폴더 전체를 IIS 사이트 경로에 복사하면 됩니다.

## 4. 복붙용 스크립트

아래 두 파일 내용을 그대로 복사해 사용하고, **맨 위 상수만** 자기 프로젝트에 맞게 바꾸면 됩니다.

### 4-1. scripts/build-for-iis.cjs

```javascript
/**
 * 서브경로 배포: VITE_BASE_PATH 로 빌드 후 dist → publish-iis/dist 복사
 * 적용 시 맨 아래 BASE_PATH_NAME 만 자기 앱 이름으로 수정 (예: MY-APP)
 */
const path = require('path')
const fs = require('fs')
const { spawnSync } = require('child_process')

const BASE_PATH_NAME = 'MY-APP'  // ← 여기만 수정 (IIS 서브경로 이름)

if (process.env.VITE_BASE_PATH === undefined) process.env.VITE_BASE_PATH = BASE_PATH_NAME
const root = path.join(__dirname, '..')
const distDir = path.join(root, 'dist')
const targetDist = path.join(root, 'publish-iis', 'dist')

console.log('VITE_BASE_PATH=', process.env.VITE_BASE_PATH)
console.log('1. vite build 실행...')
const r = spawnSync('npx', ['vite', 'build'], { cwd: root, stdio: 'inherit', env: process.env, shell: true })
if (r.status !== 0) process.exit(1)

console.log('2. dist → publish-iis/dist 복사...')
function copyDir(src, dest) {
  if (!fs.existsSync(dest)) fs.mkdirSync(dest, { recursive: true })
  for (const name of fs.readdirSync(src)) {
    const s = path.join(src, name)
    const d = path.join(dest, name)
    if (fs.statSync(s).isDirectory()) copyDir(s, d)
    else fs.copyFileSync(s, d)
  }
}
if (fs.existsSync(targetDist)) fs.rmSync(targetDist, { recursive: true })
copyDir(distDir, targetDist)
const pathNote = process.env.VITE_BASE_PATH ? `서브경로 /${process.env.VITE_BASE_PATH}/` : '루트(/)'
fs.writeFileSync(path.join(targetDist, '배포용-서브경로빌드.txt'), `${pathNote} 로 빌드됨.\n${new Date().toISOString()}`, 'utf8')
console.log('완료. npm run prepare-publish-iis 실행 후 publish-iis 폴더를 서버에 복사하세요.')
```

### 4-2. scripts/prepare-publish-iis.cjs

```javascript
/**
 * server/ → publish-iis/server/ 복사 후 npm install --omit=dev
 * 적용 시 SERVER_DIR_NAME 이 server 가 아니면 수정 (기본: server)
 */
const path = require('path')
const fs = require('fs')
const { spawnSync } = require('child_process')

const SERVER_DIR_NAME = 'server'  // ← 서버 폴더 이름 (보통 server)

const root = path.join(__dirname, '..')
const serverDir = path.join(root, SERVER_DIR_NAME)
const targetDir = path.join(root, 'publish-iis', SERVER_DIR_NAME)

const SKIP_DIR_NAMES = new Set(['node_modules', '.git', 'uploads'])
function copyRecursive(src, dest, skipDirNames = SKIP_DIR_NAMES) {
  const stat = fs.statSync(src)
  if (stat.isDirectory()) {
    if (skipDirNames.has(path.basename(src))) return
    if (!fs.existsSync(dest)) fs.mkdirSync(dest, { recursive: true })
    for (const name of fs.readdirSync(src)) {
      copyRecursive(path.join(src, name), path.join(dest, name), skipDirNames)
    }
  } else {
    if (path.basename(src).endsWith('.db')) return
    const destDir = path.dirname(dest)
    if (!fs.existsSync(destDir)) fs.mkdirSync(destDir, { recursive: true })
    fs.copyFileSync(src, dest)
  }
}

const distIndex = path.join(root, 'publish-iis', 'dist', 'index.html')
if (!fs.existsSync(distIndex)) {
  console.warn('⚠ publish-iis/dist/index.html 없음. 먼저 npm run build:iis (또는 npm run deploy:iis) 실행하세요.')
}
console.log('1. ' + SERVER_DIR_NAME + '/ → publish-iis/' + SERVER_DIR_NAME + '/ 복사...')
copyRecursive(serverDir, targetDir)
console.log('2. publish-iis/' + SERVER_DIR_NAME + ' 에서 npm install --omit=dev 실행...')
const r = spawnSync('npm', ['install', '--omit=dev'], { cwd: targetDir, stdio: 'inherit', shell: true })
if (r.status !== 0) process.exit(1)
console.log('완료. publish-iis 폴더 전체를 서버에 복사하면 됩니다.')
```

## 5. 복붙용 web.config (IIS + iisnode)

`publish-iis/web.config` 로 저장. Node 진입 파일이 `server/index.js` 가 아니면 `server/index.js` 부분을 해당 경로로 바꾸세요.

```xml
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <system.webServer>
    <defaultDocument enabled="false" />
    <security>
      <requestFiltering>
        <verbs allowUnlisted="true">
          <add verb="GET" allowed="true" />
          <add verb="POST" allowed="true" />
          <add verb="PUT" allowed="true" />
          <add verb="DELETE" allowed="true" />
          <add verb="PATCH" allowed="true" />
        </verbs>
      </requestFiltering>
    </security>
    <handlers>
      <add name="iisnode" path="server/index.js" verb="*" modules="iisnode" />
    </handlers>
    <rewrite>
      <rules>
        <rule name="Node" stopProcessing="true">
          <match url=".*" />
          <action type="Rewrite" url="server/index.js" />
        </rule>
      </rules>
    </rewrite>
    <iisnode
      nodeProcessCommandLine="&quot;C:\Program Files\nodejs\node.exe&quot;"
      node_env="production" />
  </system.webServer>
</configuration>
```

## 6. 서버 쪽에서 해 줄 일 (Node가 base 경로 아는 방법)

서브경로(예: `/MY-APP`)로 배포할 때, Node(Express)가 이 경로를 알아야 하면 환경 변수로 넘깁니다.

- **IIS**  
  `web.config` 의 `iisnode` 안에 다음 추가 (서브경로일 때만, 루트 배포면 value 빈 문자열):

```xml
<environmentVariables>
  <environmentVariable name="BASE_PATH" value="/MY-APP" />
</environmentVariables>
```

- **Express 예시**  
  `const basePath = process.env.BASE_PATH || ''`  
  정적 파일: `app.use(basePath + '/', express.static('dist'))`  
  API: `app.use(basePath + '/api', apiRouter)`  
  등으로 사용.

## 7. 정리: 다른 사람에게 줄 파일

이 방식으로 **다른 사람이 자기 프로젝트에 적용**하려면, 아래만 전달하면 됩니다.

| 전달할 것 | 용도 |
|-----------|------|
| **이 문서 전체** (`빌드배포-다른프로젝트에-적용하기.md`) | 방법 설명 + 4·5절 복붙용 스크립트·web.config |
| (선택) 이 문서만 PDF/메일로 보내도 됨 | 받는 사람이 3~5절 따라 하며 자기 프로젝트에 복사 |

받는 사람은:

1. 자기 프로젝트 루트에 `scripts` 폴더 생성
2. 이 문서 4절에서 `build-for-iis.cjs`, `prepare-publish-iis.cjs` 복사 후 **BASE_PATH_NAME**, **SERVER_DIR_NAME** 만 수정
3. `publish-iis` 폴더 만들고 5절 **web.config** 복사 (필요 시 path 수정)
4. `vite.config` 에 base 설정 (2절)
5. `package.json` 에 3-4절 스크립트 추가
6. `npm run deploy:iis` 실행 후 `publish-iis` 를 서버에 복사

이렇게 하면 **지금 쓰는 빌드·배포 방식**을 다른 사람도 자기 Cursor 프로젝트에 그대로 적용할 수 있습니다.
